# MMLib.OpenApiForYarp Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a complete, publishable v1.0.0 of MMLib.OpenApiForYarp — a .NET library that aggregates downstream OpenAPI docs onto a YARP gateway, with Scalar/SwaggerUI adapters, an extensible transformer pipeline, service discovery, sample app, tests, NUKE build, and CI/CD.

**Architecture:** UI-agnostic core (`MMLib.OpenApiForYarp`) that reads YARP config at runtime, fetches each downstream OpenAPI doc, runs it through an ordered transformer pipeline (built-ins = path rewrite → security propagation → published-paths filter, all swappable), and serves per-cluster + merged JSON. UI adapters (`.Scalar`, `.SwaggerUI`) are separate packages. Multi-target net8.0+net10.0.

**Tech Stack:** Yarp.ReverseProxy 2.3.0, Microsoft.OpenApi 3.7.0, Scalar.AspNetCore 2.16.3, Microsoft.Extensions.ServiceDiscovery 10.7.0, xUnit v3 + Verify.XunitV3 + Shouldly, NUKE 10.1.0, GitHub Actions.

**Spec:** `docs/superpowers/specs/2026-06-13-openapi-for-yarp-design.md`

**Public-API discipline:** only Configuration options, the three transformer interfaces + contexts, `IOpenApiForYarpBuilder`, the `AddOpenApiForYarp`/`Map*ForYarp` extensions, and a small `IClusterDocumentSource` (for UI adapters to enumerate documents) are `public`. ### Task 3.2: `YarpPathRewriter` — rebase downstream path → gateway path

**Files:** `PathTransformation/YarpPathRewriter.cs`; Test `PathTransformation/PathTransformScenariosTests.cs`.

Algorithm (per spec §7): from a route's `Match.Path` (gateway template → `Mprefix`) and the path transform entry (downstream template → `Pprefix`), for each downstream path `D`: if `D` starts with `Pprefix`, `remainder = D[Pprefix.Length..]`; `G = Mprefix + remainder`. Path params + catch-all remainders preserved verbatim. Supported transform keys: `PathPattern`, `PathPrefix`, `PathRemovePrefix`, `PathSet`, none.

Transform → `Pprefix` resolution:
- `PathPattern: P` → `Pprefix` = `YarpPathTemplate(P).StaticPrefix`.
- `PathPrefix: /x` → downstream receives `/x` + incoming-remainder; so the downstream path begins with `/x` ⇒ `Pprefix = /x`.
- `PathRemovePrefix: /x` → downstream had `/x` removed ⇒ gateway path must re-add it ⇒ `Mprefix = MatchStaticPrefix`, `Pprefix = ""`, then prepend nothing extra (the Match prefix already contains the upstream prefix). Concretely: `G = MatchStaticPrefix + D`. (Add a focused test.)
- `PathSet: /x` → fixed single path; map the single operation set.
- none → passthrough: `G = D` (when Match is root) else `G = MatchStaticPrefix + D`.

- [ ] **Step 1 (scenario tests — one [Fact]/[Theory] per spec scenario):** SimplePrefix_SingleRoute, CatchAll_SingleRoute, MultipleRoutes_SameCluster, PathRemovePrefix_Transform, PathPattern_WithParameters, NoTransform_Passthrough, VirtualDirectory_NestedPath, MultipleTransforms_SameRoute (header/query transforms must NOT change paths). Each builds a `RouteConfig` + downstream path set and asserts the rewritten gateway paths. Example:

```csharp
[Fact]
public void SimplePrefix_SingleRoute()
{
    var route = new RouteConfig {
        RouteId = "products", ClusterId = "products-cluster",
        Match = new RouteMatch { Path = "/api/products/{**catch-all}" },
        Transforms = new[] { new Dictionary<string,string>{["PathPattern"]="/products/{**catch-all}"} }
    };
    var rewriter = new YarpPathRewriter();
    var map = rewriter.BuildPathMap(route, new[]{ "/products/{id}", "/products" });
    map["/products/{id}"].ShouldBe("/api/products/{id}");
    map["/products"].ShouldBe("/api/products");
}
```

- [ ] **Step 2:** fail. **Step 3:** implement `BuildPathMap(RouteConfig route, IEnumerable<string> downstreamPaths) → IReadOnlyDictionary<string,string>` + a `RewriteDocumentPaths(OpenApiDocument, IReadOnlyList<RouteConfig>)`. **Step 4:** pass all scenarios. **Step 5:** commit.
