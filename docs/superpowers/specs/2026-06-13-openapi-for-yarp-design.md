# MMLib.OpenApiForYarp — Design Spec (v1.0.0)

**Date:** 2026-06-13
**Author:** Milan Martiniak (GitHub: Burgyn)
**License:** MIT
**Status:** Approved for implementation

## 1. Summary

A .NET open-source library that aggregates OpenAPI documentation from downstream
microservices and exposes it on a [YARP](https://microsoft.github.io/reverse-proxy/)
reverse-proxy gateway. Spiritual successor to **MMLib.SwaggerForOcelot** (3.8M NuGet
downloads), rebuilt for the modern .NET stack: YARP instead of Ocelot, **Scalar** as the
default UI, and `Microsoft.OpenApi` as the object model / engine (never Swashbuckle in
the core).

The library reads each downstream service's OpenAPI document at runtime, rewrites its
paths to match how the gateway exposes them (derived from YARP route config + transforms),
optionally filters/merges/propagates security, and serves the result at
`/openapi/{cluster}.json` (+ optional merged `/openapi/all.json`), wired into a UI.

## 2. Goals & non-goals

**In scope for v1.0.0 (all confirmed):**

1. Path-prefix transformation driven by YARP `RouteConfig.Transforms`.
2. Per-service OpenAPI endpoint `GET /openapi/{cluster}.json`.
3. Scalar UI integration (default), one sidebar tab per cluster.
4. Optional merged document `GET /openapi/all.json`.
5. `AddOnlyPublishedPaths` filter + **regex include/exclude** per cluster.
6. Security-scheme propagation with dedup + per-cluster override.
7. **Service discovery** integration (`Microsoft.Extensions.ServiceDiscovery`) + **Aspire** sample.
8. First-class **transformation pipeline** — built-ins are themselves swappable transformers.
9. **SwaggerUI** adapter (opt-in) in addition to Scalar.

**Out of scope for v1 (document as known limitations):**

- Request aggregation.
- Dynamic config reload.
- Authenticated "Try it out".

## 3. Key decisions (resolved during brainstorming)

| Decision | Choice | Rationale |
|---|---|---|
| Target frameworks | **multi-target `net8.0` + `net10.0`** | net8 LTS for adoption + net10 for modern stack |
| OpenAPI object model | **single `Microsoft.OpenApi` 2.x for both TFMs** (fallback 1.6.x if 2.x lacks net8) | library uses only the object model + JSON reader/writer, NOT `Microsoft.AspNetCore.OpenApi` transformer types → no `#if` needed |
| Pipeline model | **everything via pipeline** — built-ins registered first, fully reorder/remove/replace-able | matches spec §8; max extensibility |
| Core package name | `MMLib.OpenApiForYarp` (no `.Core`); UI postfix `.Scalar` / `.SwaggerUI` | cleanest for consumers, matches brand |
| Test framework | **xUnit + Verify (snapshots) + Shouldly** | Verify replaces manual golden files; Shouldly/AwesomeAssertions avoid FluentAssertions v8 commercial license |
| Build system | **NUKE** (`build/Build.cs`) | C#-native build; generates GH Actions via `[GitHubActions]` |
| CI/CD | GH Actions: CI (build+test) + Publish (push to main) | see §11 |
| Versioning | **SemVer**, single source in `Directory.Build.props`; Publish target no-ops if version already on NuGet, else pack+push+tag `v{version}` | "publish on main, skip if version unchanged" |
| Path transform semantics | **correct real YARP semantics**, not the spec's literal (and technically wrong) example | see §7 |
| OpenAPI output version | preserve source doc version where sensible; default OpenAPI 3.0 for merged | interop |
| NuGet publish | `.csproj` packable; actual publish gated behind version bump on main | |

## 5. Core components (`MMLib.OpenApiForYarp`)

### 5.1 Configuration model
- `YarpOpenApiOptions` (root): `Dictionary<string, YarpOpenApiClusterOptions> Clusters`,
  `bool MergeDocuments`, `MergedDocumentOptions MergedDocument` (Title/Version/Description),
  `TimeSpan CacheDuration` (default 60s), `TimeSpan FetchTimeout` (default 30s),
  `string DocumentEndpointPathPattern` (default `/openapi/{cluster}.json`).
- `YarpOpenApiClusterOptions` (per cluster): `string? Title`, `string OpenApiPath`
  (default `/openapi/v1.json`), `bool AddOnlyPublishedPaths`, `string[]? IncludePaths`,
  `string[]? ExcludePaths` (regex), `string? SecurityScheme` (override).
- Bound from the `YarpOpenApi` config section; also fully configurable in code.

### 5.2 Downstream OpenAPI provider — `IDownstreamOpenApiProvider`
- `Task<OpenApiDocument?> GetDocumentAsync(string clusterName, CancellationToken ct)`.
- Resolves destination address via `IServiceDestinationResolver`, builds
  `{address}{OpenApiPath}`, fetches via `IHttpClientFactory` named client, parses JSON to
  `OpenApiDocument` using the Microsoft.OpenApi reader.
- Caching: cache-aside over `IMemoryCache`, TTL = `CacheDuration`. Keyed by cluster.
- `FetchTimeout` applied via `CancellationTokenSource`. `ILogger` on fetch/parse failures
  (return null + log, endpoint responds 502/empty as appropriate).

### 5.3 Destination resolver — `IServiceDestinationResolver`
- Default `StaticDestinationResolver`: returns `Destination.Address` verbatim.
- `ServiceDiscoveryDestinationResolver`: registered only when
  `Microsoft.Extensions.ServiceDiscovery` is present in DI; uses `IServiceEndpointResolver`
  to resolve logical addresses (e.g. `https://products-service`) to concrete endpoints.
- Detection at registration time → no breaking change for static-config users.

### 5.4 YARP config access
- Read current routes/clusters via `IProxyConfigProvider.GetConfig()`.
- Map `clusterName → (matching RouteConfig(s), ClusterConfig)`; `Transforms` read from
  `RouteConfig.Transforms` (`IReadOnlyList<IReadOnlyDictionary<string,string>>`).

## 6. Transformation pipeline (the heart) — spec §8

Three transformer interfaces, all DI-registered, executed in registration order:

```csharp
public interface IOpenApiDocumentTransformer {
    Task TransformAsync(OpenApiDocument document, OpenApiDocumentTransformerContext context, CancellationToken ct);
}
public interface IOpenApiOperationTransformer {
    Task TransformAsync(OpenApiOperation operation, OpenApiOperationTransformerContext context, CancellationToken ct);
}
public interface IOpenApiSchemaTransformer {
    Task TransformAsync(/* schema */, OpenApiSchemaTransformerContext context, CancellationToken ct);
}
```

`OpenApiDocumentTransformerContext` carries: `string ClusterName`, `RouteConfig Route`
(primary route; `IReadOnlyList<RouteConfig> Routes` for multi-route clusters),
`ClusterConfig Cluster`, `YarpOpenApiClusterOptions Options`, `IServiceProvider Services`.
Operation/schema contexts extend the document context with the operation/schema + its key.

**Executor:** runs document transformers in order. A built-in `OperationSchemaWalker`
document transformer drives the registered operation/schema transformers across the doc so
all three levels participate in one ordered pipeline.

**Built-in transformers, registered first (and removable):**
1. `PathRewriteTransformer` — rewrites paths from YARP route + transforms (§7).
2. `SecurityPropagationTransformer` — propagates `securitySchemes` (§8 below in this doc / spec §6).
3. `PublishedPathsFilterTransformer` — `AddOnlyPublishedPaths` + regex include/exclude.

**Registration API (fluent, on the builder returned by `AddOpenApiForYarp()`):**
```csharp
.AddOpenApiForYarp()
    .AddDocumentTransformer<T>()
    .AddOperationTransformer<T>()
    .AddSchemaTransformer<T>()
    .AddTransformFactory<T>()   // wires YARP ITransformFactory + IOpenApiDocumentTransformer if T implements both
    .ClearOpenApiTransformers() // drop built-ins for full custom ordering
```

`AddTransformFactory<T>()`: if `T : ITransformFactory` (YARP) it's added to YARP's transform
factories; if `T : IOpenApiDocumentTransformer` it's added to our pipeline; implementing both
wires both from one registration (the yarp-swagger `ISwaggerTransformFactory` parity).

## 7. Path transformation engine

**Correction to the AI-generated spec:** its example (`Match /api/products/{**catch-all}`
+ `PathPattern: {**catch-all}` → `/api/products/{id}`) is not valid YARP — that transform
sends `/api/products/123` downstream as `/123`. The correct config for that result is
`PathPattern: /products/{**catch-all}` (or `PathRemovePrefix: /api`). The engine implements
**correct YARP semantics**; samples/tests use realistic configs.

**Model:** YARP transforms describe incoming→downstream rewriting; for documentation we need
the inverse (downstream path → gateway-facing path). The engine interprets:
- `Match.Path` (e.g. `/api/products/{**catch-all}`) as the **gateway-facing** template; its
  static prefix before the catch-all = `Mprefix`.
- The path transform as the **downstream-facing** mapping; its static prefix = `Pprefix`:
  - `PathPattern: P` → `Pprefix` = static part of `P` before route values.
  - `PathPrefix: /x` → downstream path = `/x` + incoming-remainder ⇒ treat `Pprefix = /x`.
  - `PathRemovePrefix: /x` → downstream lost `/x` ⇒ gateway path gains it back.
  - `PathSet: /x` → fixed; map accordingly.
  - no path transform → passthrough (gateway path = downstream path).

**Algorithm (per downstream path `D`):** strip `Pprefix` from `D` to get the remainder, then
`G = Mprefix + remainder`. Path parameters (`{id}`) and catch-all remainders are preserved
verbatim. Multiple routes on one cluster → produce the union of rewritten paths.

Covered scenarios (unit tests): SimplePrefix, CatchAll, MultipleRoutes/SameCluster,
PathRemovePrefix, PathPattern with parameters, NoTransform passthrough, virtual-directory
nested path, multiple transforms on one route (path + header/query — non-path transforms
must not affect paths).

## 8. Security propagation (spec §6)
- Collect `components.securitySchemes` from each downstream doc; propagate into output.
- Dedup by scheme name: keep first, `ILogger` warning on name conflict with differing content.
- Per-cluster `SecurityScheme` override selects which scheme to apply/keep.
- No security in downstream → no scheme in output.

## 9. Document merger (spec §4)
- Merge transformed cluster docs into one `OpenApiDocument`.
- Path conflict → keep first, log warning.
- Component schema conflict (same name) → dedup; differing content → keep first + warn.
- Security schemes → dedup by name.
- Merged `info` (title/version/description) from `MergedDocument` gateway config.

## 10. Endpoints & UI adapters

- `MapOpenApiForYarp()` (core): maps `GET /openapi/{cluster}.json` (fetch→pipeline→serialize
  via Microsoft.OpenApi JSON writer) and, when `MergeDocuments`, `GET /openapi/all.json`.
- `MapScalarForYarp()` (`.Scalar`): registers Scalar (`Scalar.AspNetCore`) with every cluster
  document as a source ⇒ one sidebar tab per service. No merging by default.
- `MapSwaggerUIForYarp()` (`.SwaggerUI`): Swagger UI middleware
  (`Swashbuckle.AspNetCore.SwaggerUI` only — no Swashbuckle generation) with one endpoint per
  cluster (+ merged) in the dropdown.

Public registration surface:
```csharp
builder.Services.AddReverseProxy()
    .LoadFromConfig(config.GetSection("ReverseProxy"))
    .AddOpenApiForYarp();           // + optional .AddXxxTransformer<T>()

app.MapOpenApiForYarp();
app.MapScalarForYarp();             // or app.MapSwaggerUIForYarp();
```

## 11. Build system & CI/CD

- **NUKE** `build/Build.cs`. Targets: `Clean`, `Restore`, `Compile`, `Test`, `Pack`,
  `Publish`. Bootstrappers `build.sh`/`build.cmd`/`build.ps1`, `.nuke/` config.
- **Central Package Management** via `Directory.Packages.props`; shared metadata in
  `Directory.Build.props` (TFMs, nullable, LangVersion, version, authors, license,
  repo URL, symbols, deterministic build, `GenerateDocumentationFile`).
- **GitHub Actions generated by NUKE `[GitHubActions]`:**
  - **CI** (`ci.yml`): on push to non-main branches + PRs to main → `Test`.
  - **Publish** (`publish.yml`): on push to `main` → `Test` then `Publish`.
- **Publish/versioning:** version is single-sourced in `Directory.Build.props` (SemVer).
  The `Publish` target queries the NuGet v3 API for each package's current version; if that
  version already exists → skip (no-op). Otherwise `pack` + `dotnet nuget push` (API key from
  `NUGET_API_KEY` secret) and create git tag `v{version}`. ⇒ A real publish happens only when
  the version is bumped.

## 12. Testing strategy

**Unit (`MMLib.OpenApiForYarp.Tests`)** — fast, no I/O; real `OpenApiDocument` /
`RouteConfig` / `ClusterConfig`; fixtures loaded from `TestFixtures/`. No string comparison.
- Path-transform scenarios (§7 list).
- `AddOnlyPublishedPaths`: filters unpublished, keeps-all-when-disabled, regex include-only,
  regex exclude-partial.
- Merger: two-services no-conflict, path-conflict (warn+first), schema-conflict dedup,
  merged-info from gateway, security-schemes dedup.
- Pipeline/extensibility: custom appended after built-ins, custom mutates doc, built-in
  removable, order respected, `ITransformFactory` wires both.
- Security: propagate single, propagate two dedup, explicit override, none→none.

**Integration (`MMLib.OpenApiForYarp.IntegrationTests`)** — `WebApplicationFactory` real
in-process gateway + mock `HttpMessageHandler` returning fixture JSON.
- single-cluster Scalar registered, two-services correct paths, merged all-paths,
  AddOnlyPublishedPaths filters, custom transformer applied, **service-discovery resolves
  address** (fake `IServiceEndpointResolver`), **cache-expiry refetch** (controlled
  `TimeProvider`).
- Deserialize responses as `OpenApiDocument`; assert on paths/operations/params/security.

**Snapshots (Verify)** — for the most complex scenarios (full Petstore transform, multi-
service merge, AddOnlyPublished). Serialize output → `Verify` (JSON, property-order
independent, scrub volatile values). `.verified.*` committed, `.received.*` git-ignored;
Verify auto-detects CI and disables the diff tool.

**Fixtures (`TestFixtures/`):** `petstore.openapi.json` (3.1, realistic),
`orders.openapi.json` (nested paths/params/query), `auth-service.openapi.json` (Bearer +
ApiKey), `complex-yarp.json` (multiple routes/transforms/catch-all/header/query transforms).

## 13. Sample app (`sample/`)

- `ServiceA` (Products) + `ServiceB` (Orders): ASP.NET Core minimal APIs using
  `Microsoft.AspNetCore.OpenApi`, exposing `/openapi/v1.json`, Bearer/ApiKey security where
  relevant.
- `Gateway`: YARP + `AddOpenApiForYarp()` + Scalar; static config; Orders has
  `AddOnlyPublishedPaths: true`; merged document enabled; Bearer propagated.
- `AppHost` (Aspire) + `ServiceDefaults`: service-discovery variant resolving logical names.
- `sample/README.md`: `dotnet run` instructions for both static and Aspire flows.

## 14. Quality requirements
- Multi-target `net8.0;net10.0`; nullable enabled; `GenerateDocumentationFile`.
- All public APIs have XML doc comments.
- `ILogger<T>` in fetch failures, path/schema conflicts, merge warnings.
- `IHttpClientFactory` for all downstream HTTP; cache (60s default, configurable); fetch
  timeout (sensible default).

## 15. Risks & mitigations
- **`Microsoft.OpenApi` 2.x net8 support** (riskiest): verify first thing; if 2.x has no net8
  target, pin 1.6.x on both TFMs (still single version, no `#if`). Isolate reader/writer behind
  a thin internal helper so a version switch is localized.
- **Package net10 support** (YARP, Scalar.AspNetCore, Verify, NUKE, xUnit): pin latest stable
  that supports net10; verify on restore.
- **net8 build on net10 SDK**: requires net8 reference packs (restored via NuGet) — confirm
  restore succeeds.
- **NUKE `[GitHubActions]` regeneration drift**: commit generated YAML; treat generation as
  source of truth.
